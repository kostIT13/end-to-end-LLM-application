## Соберите end-to-end RAG-сервис на open-source русскоязычном корпусе. Сервис должен:

* Индексировать корпус документов в векторное хранилище.
* Принимать запросы пользователя через HTTP-эндпоинт.
* Делать hybrid retrieval (dense + BM25).
* Делать reranking кандидатов (cross-encoder) перед генерацией.
* Генерировать ответ с цитированием источников.
* Иметь golden eval с реальными метриками.
* Иметь базовую защиту от prompt injection.
* Быть упаковано в Docker для лёгкого запуска.
