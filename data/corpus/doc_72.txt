Federated Learning: обучение без централизации данных

Federated Learning (федеративное обучение) — это парадигма ML, где модель обучается на децентрализованных данных без их централизации. Предложена Google (2016) для Gboard. Принцип: 1) Центральный сервер рассылает текущую модель клиентам. 2) Каждый клиент обучает модель на своих локальных данных. 3) Клиенты отправляют обновления весов (градиенты) на сервер. 4) Сервер усредняет обновления (Federated Averaging, FedAvg). 5) Повторение. Преимущества: конфиденциальность данных (данные не покидают устройство), снижение коммуникационных затрат, использование данных, которые нельзя централизовать (медицина, финансы). Проблемы: неоднородность данных (non-IID), коммуникационная эффективность, безопасность (атаки отравления). Улучшения: FedProx — для неоднородных данных. Secure Aggregation — шифрование обновлений. Differential Privacy — добавление шума для защиты. Применения: медицина (обучение на данных разных клиник), мобильные устройства, финансы.
