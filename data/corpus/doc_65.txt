Git и Git Flow для ML-проектов

Git — система контроля версий, основа любого ML-проекта. Особенности Git для ML: большие файлы (датасеты, модели) не хранятся в Git (используется DVC или Git LFS). Notebooks (.ipynb) — сложны для diff'а, рекомендуется очищать вывод перед коммитом (jupyter nbconvert --clear-output). Git Flow — модель ветвления: main — стабильная версия, develop — разработка, feature/* — новые фичи, release/* — подготовка релиза, hotfix/* — срочные исправления. Для ML: experiment/* — ветки для экспериментов. Git Hooks — автоматизация: pre-commit (линтеры, форматирование), pre-push (тесты). Conventional Commits — стандарт сообщений: feat:, fix:, chore:, docs:, test:. Code Review — обязателен для ML-кода. Git LFS — для хранения больших файлов (модели, датасеты). Альтернативы: DVC (специализирован для данных и моделей). Git — обязательный инструмент для любого ML-инженера.
