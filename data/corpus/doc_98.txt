Prompt Engineering: искусство составления промптов

Prompt Engineering — это практика составления и оптимизации промптов (инструкций) для языковых моделей. Качество промпта критически влияет на качество ответа. Основные техники: Zero-shot — прямая инструкция без примеров. Few-shot — несколько примеров в промпте. Chain-of-Thought (CoT) — пошаговое рассуждение ("Let's think step by step"). Tree-of-Thought (ToT) — дерево рассуждений. Self-Consistency — несколько прогонов CoT и голосование. Role Prompting — назначение роли ("Ты эксперт по..."). System Prompt — системные инструкции. Structured Output — требование формата (JSON, XML). Negative Prompting — указание, чего не делать. Context Management — управление контекстом (релевантная информация). Prompt Chaining — последовательность промптов. ReAct — Reasoning + Acting (интеграция с инструментами). Инструменты: LangChain, LlamaIndex, PromptPerfect, Anthropic Console. Prompt Engineering — важный навык для работы с LLM, хотя с улучшением моделей потребность в сложных промптах снижается.
