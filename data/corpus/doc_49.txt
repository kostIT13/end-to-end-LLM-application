CI/CD для машинного обучения

CI/CD (Continuous Integration / Continuous Deployment) для ML — это автоматизация тестирования, сборки и развертывания ML-моделей. Отличия от традиционного CI/CD: данные меняются со временем (data drift), модели могут деградировать, нужны автоматические тесты качества. CI-этапы: линтинг кода, запуск unit-тестов, тестирование пайплайнов данных, проверка качества модели на валидационном наборе, проверка на data drift. CD-этапы: упаковка модели в Docker-образ, деплой в staging, A/B тестирование, promotion в production, мониторинг. Инструменты: GitHub Actions, GitLab CI, Jenkins, CircleCI для CI. Для ML-specific: CML (Continuous Machine Learning) — инструмент от DVC для CI/CD ML. Kubeflow Pipelines, Airflow — для оркестрации пайплайнов. Важно: model validation — проверка, что новая модель не хуже текущей в продакшене. Rollback — возможность отката к предыдущей версии. Feature store — централизованное хранилище признаков.
