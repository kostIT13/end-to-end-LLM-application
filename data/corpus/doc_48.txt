DVC: управление версиями данных и моделей

DVC (Data Version Control) — это open-source инструмент для версионирования данных и ML-моделей, работающий поверх Git. В отличие от Git, который не предназначен для больших файлов, DVC хранит метаданные в Git, а сами данные — в удаленном хранилище (S3, GCS, Azure Blob, SSH). Основные возможности: версионирование датасетов — dvc add, dvc push, dvc pull. Пайплайны — dvc run, dvc repro — определение шагов (data processing, training, evaluation) с кэшированием результатов. Метрики — dvc metrics — отслеживание метрик моделей. Эксперименты — dvc exp — параллельные эксперименты с разными гиперпараметрами. DVC позволяет воспроизводить любой эксперимент из прошлого, переключаясь между Git-коммитами и запуская dvc checkout. Интеграция с MLflow, CML (Continuous Machine Learning). DVC — стандарт для версионирования данных в ML-проектах, альтернатива — Git LFS, LakeFS.
