RAG: Retrieval-Augmented Generation

RAG (Retrieval-Augmented Generation) — это архитектура, которая комбинирует поиск информации (retrieval) с генерацией текста (generation). RAG позволяет LLM отвечать на вопросы, используя внешние источники знаний, что решает проблему галлюцинаций и устаревших знаний. Архитектура: Indexing — индексация документов (chunking → embedding → vector store). Retrieval — поиск релевантных чанков по запросу (dense retrieval, hybrid search). Generation — LLM генерирует ответ на основе найденных документов. Компоненты: Chunker — разбиение документов на чанки. Embedder — модель для эмбеддингов (text-embedding-3, nomic-embed-text). Vector Store — хранилище эмбеддингов (Pinecone, Weaviate, pgvector). Retriever — поиск (Dense, Sparse, Hybrid). Reranker — переранжирование результатов. LLM — генерация ответа. Продвинутые техники: HyDE (Hypothetical Document Embeddings) — генерация гипотетического документа для поиска. Multi-Query Retrieval — несколько вариаций запроса. Parent Document Retriever — поиск по чанкам, возврат целых документов. RAG — стандартная архитектура для QA-систем и чат-ботов на основе документов.
